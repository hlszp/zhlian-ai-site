# 网站优化与安全审计（2026-07-15）

## 审计范围

- 公网生产站点 `http://ai.zlinfot.com/` 的桌面端、移动端、主要交互和基础 SEO。
- React 前端、Markdown 内容管线、FastAPI Phase 2 后端、Nginx 配置、发布脚本和 GitHub Actions。
- 内容与外部链接一致性、依赖漏洞、敏感信息和未授权管理接口。

## 主要结论

生产站点当前仍为静态站点，Phase 2 后端与管理端尚未接入公网。基线检查发现移动导航缺失、键盘交互不完整、触控目标偏小、SEO 元数据不完整、重复内容和两条失效的 ISA 链接；同时发现生产打包依赖本地虚拟环境，导致 GitHub Actions 连续失败。

安全审计确认：Phase 2 的 10 个内容管理 API 原先可匿名访问，数据库连接地址会进入日志，且仓库中的生产规划曾准备在未启用 TLS 时使用 Basic Auth。这些问题已在本分支修复，并将生产流水线恢复为静态站点专用。Phase 2 继续保持手动 staging，必须在 HTTPS、生产数据库、服务用户和回滚方案准备完毕后再申请上线。

## 已实施优化

- 增加响应式移动导航、键盘可操作的知识库标签页、统一 `focus-visible` 样式和不少于 44px 的主要触控目标。
- “查看能力链”现在会滚动并聚焦到对应知识网络；筛选按钮和结果数量补充状态语义。
- 增加 canonical、Open Graph、Twitter Card、`robots.txt` 和 `sitemap.xml`。
- 页脚统一为简洁的 `ZHILIAN © <year>`，不扩展共识名称或宣传口号。
- 删除重复的 HAI 条目；内容校验新增重复 ID 和重复来源摘要检查；修复失效 ISA 链接。
- 所有文章、分类和标签管理 API 增加 HTTP Basic 鉴权；生产配置拒绝弱口令和默认数据库凭据。
- 停止输出数据库连接地址，并从版本库移除缓存、测试数据库等生成文件。
- 生产包默认只包含静态站点；GitHub Actions 不再使用仓库内 `.venv`，生产部署不再调用后端部署或 `sudo`。仓库 Nginx 模板会让 `/api` 与 `/admin` 明确返回 404，但普通静态部署不会自动应用该模板，服务器配置变更仍需单独批准。

## 验证要求

合并前必须通过：内容校验、前端构建、TypeScript 类型检查、后端 pytest、部署脚本测试、静态发布包校验、依赖审计、桌面与移动端浏览器回归。CI/CD 的权威操作说明见仓库根目录 `README.md` 和 `CLAUDE.md`。

## 剩余风险与下一步

- 公网站点仍是 HTTP；TLS 是启用任何登录或管理能力前的硬性前置条件。
- Phase 2 后端尚未进行独立的生产数据库备份恢复、服务监控、限流和 staging 回滚演练。
- 当前 PR 合并后会自动触发静态生产部署；后端部署仍需单独方案和明确批准。
